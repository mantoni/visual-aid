# Decisions

## Purpose

This directory stores architectural decision records in markdown.

Each decision record should:

- have a stable number
- describe the context
- state the decision
- explain the consequences

## Naming

Use the format `NNNN-short-title.md`, for example:

- `0001-markdown-first-agent-workflow.md`
- `0002-payload-schema-v1.md`

## When To Add A Decision Record

Add a new record when a change meaningfully affects:

- system architecture
- product behavior
- workflow expectations
- integration contracts
- storage or transport formats

## Status

Each decision record should include a status such as:

- Proposed
- Accepted
- Superseded

If a decision is replaced later, keep the old record and mark it as superseded rather than deleting it.
